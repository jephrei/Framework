Github Download Issues

This is a minimal framework (not necessarily small but at the moment it's ~12k minified) that can be applied to multiple use cases. It isn't meant to assume the final product so this is for people to use as a foundation under their own code. This is a personal framework that is in the alpha stages of development so things will change and/or break. I would love this to get used, but I highly suggest you make your own.

The Grid

It can be nested, the columns used in any combination, reversed in mobile, with or without gutters. It's a damn fine grid.

Columns

The system is based on fractions. 1/2, 2/3, 5/8, etc so the total number of columns is irrelevant. There's no margin trickery here either; a column with the class .span-1-2 is exactly 50% wide.

Of course, the columns stack on devices with smaller screens. Add .no-stack to a specific row to prevent this.

Offset

Offset or "skip" columns by adding the .skip-* class. For every column width there is an equal offset. For example, because .span-4-7 exists, so does .skip-4-7.

.row > .col.skip-*

.col.span-1-2.skip-1-8

Gutters

The gutters are exactly 2em for readability. They can be added by adding the .spaced class to a row. Pseudo classes or extra .first and .last type classes aren't required here as each column is perfectly equal.

.row.spaced > .col

Reverse for Stacked View

Using the .reverse class on a row will reverse the order of the columns in desktop, but stack in order on smaller devices.

.row.reverse > .col

2nd in desktop, 1st in mobile
1st in desktop, 2nd in mobile

Rows & Containers

Every column needs to be placed within a .row. Rows are always full width and are resitriced in width (if desired) by the .container class. This controls the overall width of the page.

The container can wrap the entire page, wrap sections, or can be within any full viewport width element like a typical header or footer.

Typography

The font families chosen are serif, sans-serif, and monospace...that's it. Browsers and their environment decide on best looking font available if it's not specified. The font sizes are em based so they scale with the body font-size percentage which, in turn, changes with media queries.

Headers

The header sizes have been reset so that h6 is equal in size to the body font size. Dangerous? Possibly. It's an experiment so i may undo it depending on how it works in practice.

Elements

Images

Here are a few helper classes for images in the stacked view. Throw these on a DIV with the image inside. .crop-stacked will crop off the top and bottom 25% leaving your image 50% tall on smaller devices. Good so large height images don't take up too much space on mobile devices. .flush-stacked will break your image out to full width in stacked view. Combine them for both effects.

.crop-stacked img

.flush-stacked img

.flush-stacked.crop-stacked img
